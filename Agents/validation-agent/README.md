# Validation Agent PRD & Technical Spec

## 1. Overview
The Validation Agent acts as a safety gatekeeper. It receives the structured JSON execution plan generated by the master-agent and validates it against a set of security, policy, and configuration rules **before** the worker-agent executes anything on AWS. Think of it as the "Are you sure?" layer that prevents costly mistakes.

## 2. Tech Stack & Environment
- **Framework:** Python + FastAPI.
- **LLM SDK:** `google-genai` (Gemini), `openai`, or LangChain (optional — can also be rule-based).
- **Internal Network Port:** `8002`.

## 3. Mandatory File Structure
```text
Agents/validation-agent/
├── main.py                  # FastAPI server and route definitions
├── rules/
│   ├── policy_rules.py      # Hardcoded policy checks (e.g., blocked instance types)
│   ├── security_rules.py    # Security checks (e.g., no public S3 buckets)
├── llm/
│   ├── client.py            # Optional: LLM-based validation for edge cases
│   ├── prompts.py           # System prompt for AI-powered validation
├── models/
│   ├── schemas.py           # Pydantic models for input/output validation
├── requirements.txt         # pip install fastapi uvicorn openai/google-genai
```

## 4. System Rules
**Rule 1:** You do NOT execute any AWS commands. You only validate and approve/reject plans.
**Rule 2:** You can use a mix of hardcoded rules AND LLM-based reasoning. Example hardcoded rules:
  - Block instance types larger than `t2.medium` (cost control).
  - Block S3 buckets with public access enabled (security).
  - Ensure region is in the allowed list (`ap-south-1`, `us-east-1`).
**Rule 3:** Always return a clear `approved: true/false` with a human-readable `reason`.

## 5. API Communication Contracts

### Expose: `POST /validate`
**Input (from Backend):**
```json
{
  "service": "ec2",
  "action": "launch_instance",
  "parameters": {
    "InstanceType": "m5.4xlarge",
    "Region": "eu-west-1"
  }
}
```

**Output (to Backend) — REJECTED example:**
```json
{
  "approved": false,
  "reasons": [
    "Instance type m5.4xlarge exceeds allowed limit (max: t2.medium).",
    "Region eu-west-1 is not in the allowed regions list."
  ]
}
```

**Output (to Backend) — APPROVED example:**
```json
{
  "approved": true,
  "reasons": []
}
```
